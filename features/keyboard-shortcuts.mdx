## Input Field Handling

Shortcuts are **disabled** when typing in input fields:

```typescript
const onKeyDown = async (e: KeyboardEvent) => {
  // Ignore if typing in an input field
  if (e.target instanceof HTMLInputElement || e.target instanceof HTMLTextAreaElement) {
    return;
  }

  // Process shortcuts
  if (e.ctrlKey && e.key.toLowerCase() === 's') {
    e.preventDefault();
    setView('search');
  }
};
```

<Tip>
  This prevents shortcuts from interfering when you're typing search queries or AI DJ messages.
</Tip>

## Implementation Details

### Event Listener Setup

```typescript
useEffect(() => {
  if (!firstBootDone) return;

  const onKeyDown = async (e: KeyboardEvent) => {
    // Ignore if typing in an input field
    if (e.target instanceof HTMLInputElement || e.target instanceof HTMLTextAreaElement) {
      return;
    }

    // Escape: Back to app
    if (e.key === "Escape" && view !== "app") {
      e.preventDefault();
      setView("app");
      return;
    }

    // Ctrl shortcuts
    if (e.ctrlKey && !e.shiftKey && !e.altKey) {
      switch (e.key.toLowerCase()) {
        case "s": // Ctrl+S: Search
          e.preventDefault();
          setView("search");
          break;
        case "p": // Ctrl+P: Playlists
          e.preventDefault();
          setView("playlist");
          break;
        case "e": // Ctrl+E: Settings
          e.preventDefault();
          setView("settings");
          break;
        case "m": // Ctrl+M: Volume
          e.preventDefault();
          setView("volume");
          break;
        case "d": {
          // Ctrl+D: AI DJ (if available)
          e.preventDefault();
          const settings = await readSettings();
          const hasAI =
            getActiveAIProvider(settings.ai_providers, settings.active_ai_provider) !== null;
          if (hasAI) {
            setView("aidj");
          }
          break;
        }
      }
    }
  };

  document.addEventListener("keydown", onKeyDown);
  return () => document.removeEventListener("keydown", onKeyDown);
}, [firstBootDone, view]);
```

### Conditional AI DJ Shortcut

The AI DJ shortcut only works when a provider is configured:

```typescript
case "d": {
  e.preventDefault();

  const hasAI = getActiveAIProvider(
    settings.ai_providers,
    settings.active_ai_provider
  ) !== null;
  if (hasAI) {
    setView("aidj");
  }
  break;
}
```

## Shortcuts Configuration File

MiniFy uses a `shortcuts.json` file for configuration:

```bash
~/workspace/source/apps/desktop/src/config/shortcuts.json
```

<Note>
  The shortcuts configuration file is currently empty (`{}`), indicating that shortcuts are hardcoded in the UI layer. Future versions may support custom shortcut mapping via this file.
</Note>

## Customizing Shortcuts

To customize shortcuts, modify the keyboard event handler in `src/ui/index.tsx`:

```typescript
// Example: Change search to Ctrl+F instead of Ctrl+S
case "f": // Changed from "s"
  e.preventDefault();
  setView("search");
  break;
```

<Warning>
  Modifying keyboard shortcuts requires rebuilding the application.
</Warning>

## Platform Considerations

<Tabs>
  <Tab title="Windows/Linux">
    All shortcuts use the `Ctrl` key as the modifier.
    
    - `Ctrl+S` - Search
    - `Ctrl+P` - Playlists
    - `Ctrl+E` - Settings
  </Tab>
  
  <Tab title="macOS">
    Future versions may support `Cmd` key mapping for macOS:
    
    - `Cmd+S` - Search
    - `Cmd+P` - Playlists
    - `Cmd+E` - Settings
    
    <Note>
      Currently, macOS users should use `Ctrl` instead of `Cmd`.
    </Note>
  </Tab>
</Tabs>

## Accessibility

Keyboard shortcuts improve accessibility by:

- Enabling mouse-free navigation
- Providing consistent, predictable patterns
- Respecting input field focus
- Displaying shortcuts in context menus

<Tip>
  All shortcuts follow standard conventions (e.g., `Ctrl+S` for search, `Escape` to close) to match user expectations.
</Tip>
